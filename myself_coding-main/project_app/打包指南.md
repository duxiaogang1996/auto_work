# Windows 下打包项目为 exe 的指南

## 推荐方式：从 GitHub Release 直接下载打包好的 exe（推荐！）

本项目已配置 GitHub Actions 自动构建，当你推了一个带版本标签的提交到 GitHub 仓库后，GitHub 会自动在云端构建 Windows exe，你可以直接下载使用，无需在本地打包。

**使用步骤：**

1. 将代码推送到 GitHub
2. 创建一个版本标签（例如 `git tag v1.0.0` 然后 `git push origin v1.0.0`）
3. GitHub Actions 会自动构建出 `自动化控制台.zip`
4. 进入 GitHub 页面的 "Release" 下载 zip 压缩包
5. 解压后双击 `自动化控制台/自动化控制台.exe` 即可使用

## 备选方式：在 Windows 本地手动打包

如果需要本地测试或不使用 GitHub，可以按照以下步骤在你的 **Windows 电脑** 上手动打包：

为了在 Windows 上将此项目打包为一个独立的 `.exe` 文件（包含前端网页和后台接口），你需要使用 `PyInstaller`。main.py 已经过修改，使其能够正确识别打包后的静态资源（HTML/CSS/JS）路径，并且在启动程序时会自动打开浏览器。

请在你的 **Windows 电脑** 上按照以下步骤操作：

## 第一步：准备环境

1. 将整个 `project_app` 文件夹复制到你的 Windows 电脑上。
2. 确保 Windows 电脑上已经安装了 Python（建议版本 3.10 或以上）。
3. 打开命令提示符（cmd）或 PowerShell，进入 `project_app` 目录：
   ```cmd
   cd 你的路径\project_app
   ```

## 第二步：安装依赖

在 `project_app` 目录下，运行以下命令安装项目所需的库以及打包工具：

```cmd
pip install -r requirements.txt
```
*(这会自动安装 Flask、requests 以及 PyInstaller)*

## 第三步：执行打包命令

在 `project_app` 目录下，运行以下 `PyInstaller` 命令。这条命令非常关键，它告诉 PyInstaller 去哪里找前端的 HTML 和静态文件，并将它们打包进 exe 中：

```cmd
pyinstaller --name "自动化控制台" --noconfirm --onedir --windowed --add-data "ui;ui/" --paths "." main.py
```

**命令参数解释：**
- `--name "自动化控制台"`：生成的 exe 文件名。
- `--noconfirm`：在重新打包时自动覆盖现有的输出目录。
- `--onedir`：打包成一个文件夹（包含 exe 和一些依赖的 dll）。**强烈推荐使用这个而不是 `--onefile`**，因为带有前端模板的 Flask 启动速度在 `--onedir` 模式下快很多。
- `--windowed`：运行时不显示黑色的命令行控制台窗口（后台静默运行）。
- `--add-data "ui;ui/"`：将 `ui` 文件夹（包含 `templates` 和 `static`）打包进最终程序中。（注意：在 Windows 下路径分隔符用分号 `;`，Mac/Linux 下用冒号 `:`）。
- `--paths "."`：将当前目录作为包搜索路径，确保本地的自定义模块（如 `core` 文件夹下的代码）能够被正确打包进去，避免出现 `ModuleNotFoundError`。

## 第四步：获取和运行执行文件

1. 打包完成后，你会看到 `project_app` 目录下多出了 `build` 和 `dist` 两个文件夹。
2. 进入 `dist` 文件夹，你会看到一个名为 `自动化控制台` 的文件夹。
3. 双击运行 `自动化控制台` 文件夹里面的 `自动化控制台.exe`。
4. 程序启动后（没有黑框），它会**自动在你的默认浏览器中打开** `http://127.0.0.1:5000`，你就可以直接使用了。

## 如何关闭程序？

因为使用了 `--windowed` 隐藏了控制台，你可以通过以下方式关闭它：
1. 打开 Windows 的“任务管理器”（Ctrl+Shift+Esc）。
2. 在进程列表中找到 `自动化控制台.exe`，右键点击“结束任务”。
